    创作

    创作可以分解为收集(数据)，整理(过滤信息)，内化(吸收知识)，out(输出智慧)。

    收集

    收集数据 目的：明确清晰 -> 量化 可表达

    关于对象

    • 喜欢的
    • 经典的
    • 优秀的(多元，创意的排列组合)

    关于方法

    • 对应的网站
      • 视频教程：YouTube
      • 文字教程：
        1. What <- 定义 -> Wiki
        2. How <- 怎样实现(现实操作，需要兴趣) -> Google，Blog
        3. Why -> 输出(经验，优化后的范式，元经验的增长内容，现实的优化，之后的新发展)

    想象力之外

    • 积累过的遗忘信息。
      • 入门的纪录片(旅行美食艺术)
      • 杂书

    整理

    过滤信息，二次优化目标(改善)

    关于工具(合适很重要)

    • 收集工具
      • 暂时 -> 网页收集 -> Pocket
      • 短时(不止一次) -> Bookmark
      • 长期(网站) -> Reeder(blog) Discord(便于分类搜索)
    • 整理工具
      • 短内容 -> Diigo
      • 重要(cs相关) -> TW

    内化

    吸收属于自己的知识，发现隐藏的自己。 将信息转化为知识，需要找到信息和自己连接的部分。将知识作为自我的扩展。

    方法

    • 作业实践理论，专注于How，如何实现理论在现实中的折射？
    • 改善现实，找到理论和现实相连接的管道

    Out

    知识转化为自身的智慧

    形式

    • 文字。用语言来表达DIKW的过程。
    • 程序。How的过程。
    • 作品。视听触觉 -> 美术 音乐 玩法机制 软件

    例子

    • 脚本。workflow的优化。
    • 游戏。自我的表达。艺术的组合。
